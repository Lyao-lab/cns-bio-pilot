# Presentation Timing Guidelines (quick-reference tables)

> Planning how many slides and how much time per section. This file is the lookup tables; for slide design see `references/figure_design.md` + `figure_aesthetics.md` + `figure_layout.md`.

## Core rule
**~1 slide per minute** baseline. Adjust: complex/data slides 2–3 min; title/dividers 15–30 s; key-result slides 2–4 min. Animation build-steps count as extra slides. **Spend 40–50% of time on results. Never skip conclusions.**

## Slide count by talk length

| Duration | Total slides | Title/Intro | Methods | Results | Discussion | Conclusion |
|---|---|---|---|---|---|---|
| 3 min (poster spotlight) | 3-4 | 1 | 0 | 1-2 | 0 | 0-1 |
| 5 min (lightning) | 5-7 | 1-2 | 0-1 | 2-3 | 1 | 1 |
| 10 min | 10-12 | 2 | 1-2 | 4-5 | 2-3 | 1 |
| 15 min (conference) | 15-18 | 2-3 | 2-3 | 6-8 | 3-4 | 1-2 |
| 20 min | 20-24 | 3 | 3-4 | 8-10 | 4-5 | 2 |
| 30 min | 25-30 | 3-4 | 5-6 | 10-12 | 6-8 | 2 |
| 45 min (seminar) | 35-45 | 4-5 | 8-10 | 15-20 | 8-10 | 2-3 |
| 60 min (defense/invited) | 45-60 | 5-6 | 10-12 | 20-25 | 10-12 | 3-4 |

> More slides if: highly visual, many simple concepts, progressive builds. Fewer if: deep dive, complex figures, interactive discussion.

## Section time allocation (% of total)

| Talk type | Intro | Methods | Results | Discussion | Conclusion |
|---|---|---|---|---|---|
| Any length | 15-20% | 15-20% | **40-50%** | 15% | 5% |
| 15-min talk | 2-3 min | 2-3 min | 6-7 min | 2-3 min | 1 min |
| 45-min seminar | 8-10 min | 8-10 min | 18-22 min | 10-12 min | 2-3 min |

## Time per slide type

| Slide type | Time | Notes |
|---|---|---|
| Title / section divider | 15-30 s | Say title, brief transition, move on |
| Single-bullet / transition | 30-45 s | Paraphrase + 1-2 sentences |
| 3-4 bullet slide | ~1 min | 15-20 s per bullet; don't read verbatim |
| 5-6 bullet slide | ~2 min | Consider splitting |
| Equation slide | 1-2 min | Context 20s → terms 40s → implications 20-40s |
| Data viz / single figure | 2-3 min | Setup 30s → patterns 60s → finding 30s → stats 30s → interpret 30s |
| Multi-panel figure | 2-4 min | Progressive build or all-at-once; 60s/panel + 30-60s integrate |
| Table | 1-2 min | Don't read every cell; guide attention to key row |

## Self-monitor checkpoints

| Talk length | Checkpoint 1 | Checkpoint 2 | Checkpoint 3 |
|---|---|---|---|
| 5 min | 1 min (done problem) | 2.5 min (mid-results) | — |
| 15 min | 3-4 min (end intro) | 7-8 min (mid results) | 12-13 min (start conclusion) |
| 45 min | 10 min (end intro) | 20 min (mid methods) | 35 min (end results) |

## Lightning talk (5 min) exact plan
```
0:00-0:15  Title
0:15-1:00  The Problem
1:00-2:00  Your Solution
2:00-4:30  Key Result (1-2 figures)
4:30-5:00  Impact + Conclusion/Contact
```
No buffer — practice exact timing.

## Practice runs (minimum)

| Talk type | Full practice runs | Total time |
|---|---|---|
| Lightning (5 min) | 5-7 | ~3 hrs |
| Conference (15 min) | 3-5 | 4-5 hrs |
| Seminar (45 min) | 3-4 | 6-8 hrs |
| Defense (60 min) | 4-6 | 10-15 hrs |

Progression: Run 1 rough (expect to run long) → Run 2 smooth transitions → Run 3 timed to target → Run 4+ polish (record video, practice Q&A).

## If time is cut short
Keep intro (brief) → methods 30s → **one** main result 3 min → conclusion 30s. Skip secondary results/discussion. Pre-mark which slides are "must keep" vs "optional"; have 5/10/15-min versions ready.

## If given extra time
Go deeper on key results / show backup slides / extend discussion / more Q&A. Don't add filler or repeat content.

## Q&A timing
- Q&A inside your slot: reserve 20-30% (15-min talk → 3-4 min; 45-min → 10-15 min). Finish content 2-3 min early.
- Q&A after slot: use full time, finish exactly at limit, have backup slides for likely questions.
- Answers: 30-90 s each; redirect deep questions to "let's discuss after."

## Emergency recovery
- **Running over**: skip backup slides, summarize instead of detailing, cut discussion (never results, never conclusion).
- **Running under**: expand key points, add an example, slow down (no filler).
- **Better to finish early than run over.**
